1 Your Test Calls the Builder

buildHorticultureLodgePayload({ destinationCountry: 'MY' })

This function comes from test-data/commodities/horticulture.ts.

But it's not written there — it was created by a factory:

// horticulture.ts
export const { buildDefaultLodgePayload, ... } = createCommodityBuilders(DEFAULTS);
Analogy: Think of createCommodityBuilders as a cookie cutter factory. You give it a shape (horticulture defaults), and it gives you back tools (builders) that can stamp out cookies (payloads) in that shape.

The builder starts with the horticulture DEFAULTS object — all the "normal" values for a horticulture LODGE:

The DEFAULTS object also pulls two values from your .env.sit file:

3 Your Override Is Applied

You passed { destinationCountry: 'MY' }. The builder uses the o() function to merge:

// o() = "use override if provided, otherwise use default"

o('SG', 'MY')   // → 'MY' ✅ your override wins
o('V001', undefined) // → 'V001' ← default kept (you didn't override it)
o('SEA', null)     // → undefined ← field removed from payload (if you passed null)
Override ValueResultMeaning
'MY'Use 'MY'Your value replaces the default
undefined (not passed)Use default ('SG')Keep the default
nullundefinedDeliberately omit the field
Analogy: It's like filling out a form with pencil defaults. You can write over any field (override), leave it (use default), or erase it (null = remove).

6 HTTP Request Sent

The SoapClient sends the XML to the REX service via HTTP POST:

sequenceDiagram participant SC as SoapClient participant REX as REX Service
(government server) SC->>REX: POST https://online-sit.agriculture.gov.au/.../RexSubmissionService Note over SC: Headers:
Content-Type: text/xml
SOAPAction: ".../LodgeRex" Note over SC: Body: SOAP XML envelope REX-->>SC: HTTP 200 + SOAP XML response Note over REX: (or HTTP 500 if fault)

The URL comes from your .env.sit file:

REX_SUBMISSION_SERVICE_URL=https://online-sit.agriculture.gov.au/nexdoc-ws-ext-proxy/RexSubmissionService
Analogy: You put your form in an envelope, address it to the government office (URL), write "LODGE REQUEST" on the outside (SOAPAction header), and post it. They process it and mail back a response.

What You Can Do With the Result

// 1. Assert on the raw result
expect(result.success).toBe(true);
if (result.success) {
  expect(result.complianceStatus).toBe('INIT');
  expect(result.rexNumber).toBeTruthy();
}

// 2. Convert to RexState for next step
const rexState = toRexState(result as SoapSuccessResult);
// rexState = { rexNumber: 'REX0003487048', lastAmendmentTimestamp: '2026-03-27T...' }

// 3. Use rexState to build AMEND payload
const amendPayload = buildDefaultAmendPayload(rexState, { productType: 'TUR' });
const amendResult = await soapClient.amendRex(amendPayload);
